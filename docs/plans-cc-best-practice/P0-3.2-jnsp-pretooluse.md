# Plan — §3.2 Migrer TODO-JNSP vers un hook `PreToolUse` natif

> **Recommandation** : §3.2 de `analyse-claude-code-best-practice.md`
> **Priorité** : P0 · Impact 🔴 Critique · Effort Faible
> **Statut** : PLAN — aucune modification du SDD Mode

---

## 1. Intention (le POURQUOI)

Le protocole JNSP (« je ne sais pas faire » = signal valide) est aujourd'hui appliqué par `.aiad/hooks/pre-commit.sh`, qui ne se déclenche **que** sur la commande `git commit` côté shell. Si l'agent commit via un autre chemin (API, outil Bash composite, environnement headless sans hook git installé), le blocage saute. L'analyse (§2.5, §6.2) recommande d'**ajouter** un hook `PreToolUse` natif Claude Code : il s'applique **au niveau du harness**, hors-CLI git, et module sa rigueur via `${CLAUDE_EFFORT}`. On **garde** `pre-commit.sh` comme filet de sécurité (défense en profondeur).

## 2. État actuel SDD Mode

| Élément | Fichier réel | Couverture |
|---------|--------------|------------|
| Blocage `TODO-JNSP:` | `.aiad/hooks/pre-commit.sh` | git CLI uniquement |
| Bypass tracé | `AIAD_SKIP_DRIFT_CHECK=1`, `--no-verify` | existant |
| Hook harness | `templates/.claude/settings.json` → `SessionStart` seul | **pas de `PreToolUse`** |
| Convention | `CLAUDE.md` : `// TODO-JNSP: <question>` + exit codes 0/1/2 | définie |

## 3. Cible enforced

Un hook `PreToolUse` matcher `Bash`, `if: "Bash(git commit *)"`, qui :
- scanne le diff *staged* pour `TODO-JNSP:` non résolu,
- renvoie `hookSpecificOutput.permissionDecision: "deny"` + exit 2,
- ajuste le périmètre selon `${CLAUDE_EFFORT}` (en effort `max`, peut aussi scanner `git diff` non-staged).

## 4. Conception détaillée

### 4.1 Script de hook réutilisable
Extraire la logique de scan de `pre-commit.sh` vers un script Node partagé `.aiad/hooks/jnsp-scan.js` (DRY) qui :
- lit le diff (`git diff --cached --unified=0`),
- détecte `TODO-JNSP:` ajoutés (lignes `+`),
- sort `{ "decision": "deny", "reason": "...", "occurrences": [...] }` en JSON sur stdout,
- exit 2 si occurrence, exit 0 sinon.

`pre-commit.sh` appelle ce même script → une seule source de vérité.

### 4.2 Déclaration du hook (settings émis)
```jsonc
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "if": "Bash(git commit *)",
        "hooks": [{
          "type": "command",
          "command": "node ${CLAUDE_PROJECT_DIR}/.aiad/hooks/jnsp-scan.js --effort ${CLAUDE_EFFORT}",
          "statusMessage": "Vérification TODO-JNSP…"
        }]
      }
    ]
  }
}
```

### 4.3 Modulation par effort
- `low`/`medium` : scan staged uniquement (comportement actuel).
- `high`/`max`/`xhigh` : scan staged + working tree + rappel des JNSP en suspens dans les SPECs liées.

## 5. Étapes d'implémentation séquencées

1. **Spec** : SPEC-NNN-jnsp-hook (EARS recommandé).
2. Extraire `jnsp-scan.js` depuis `pre-commit.sh` ; faire pointer `pre-commit.sh` dessus (zéro régression).
3. Ajouter le bloc `PreToolUse` dans `templates/.claude/settings.json` (fusion avec `SessionStart` existant).
4. Brancher `--effort` (lecture de `${CLAUDE_EFFORT}`, défaut `medium`).
5. Logger chaque exécution dans `.aiad/metrics/hook-runs.jsonl` (cohérent avec l'existant).
6. Mettre à jour la doc JNSP (`CLAUDE.md`/`AGENTS.md`) : « blocage à 2 niveaux (git + harness) ».

## 6. Fichiers touchés / créés

- **Créés** : `.aiad/hooks/jnsp-scan.js`, `templates/.aiad/hooks/jnsp-scan.js` (template émis).
- **Modifiés** : `.aiad/hooks/pre-commit.sh` (délègue au script), `templates/.claude/settings.json`, `lib/hooks.js` (installation du nouveau hook), section JNSP de `emit-rules`.

## 7. Critères d'acceptation (EARS)

- **WHEN** l'agent tente `git commit` via Bash, **the system SHALL** scanner le diff pour `TODO-JNSP:` non résolu **avant** l'exécution.
- **IF** au moins un `TODO-JNSP:` non résolu est présent, **the system SHALL** refuser l'outil avec `permissionDecision: "deny"` et exit code 2.
- **WHERE** `${CLAUDE_EFFORT}` ≥ `high`, **the system SHALL** étendre le scan au working tree.
- **the system SHALL** continuer à bloquer via `pre-commit.sh` même si le hook harness est absent (défense en profondeur).

## 8. Tests & vérification

- Test unitaire `jnsp-scan.js` : diff avec `TODO-JNSP:` → exit 2 + JSON ; diff propre → exit 0.
- Test de non-régression : `pre-commit.sh` produit le même verdict qu'avant (même script sous-jacent).
- Test harness headless (`claude --print`) : commit bloqué hors git hooks.

## 9. Risques, dépendances, rollback

- **Risque** : double blocage (git + harness) perçu comme redondant → message clair indiquant la source.
- **Dépendance** : §3.4 (format JSON/exit codes commun).
- **Rollback** : toggle `disablePreToolUseHook` dans `hooks-config.json` ; `pre-commit.sh` reste actif.

## 10. Effort & découpage en SPECs

- SPEC unique, **Effort Faible**. Réutilise directement le pattern de §3.3 (même mécanique de hook bloquant). À livrer tôt — sert de modèle aux autres hooks P0.
