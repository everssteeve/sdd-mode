Glossaire utile

Intent Statement

Le POURQUOI capturé avant toute spec. Document `INTENT-NNN.md` qui exprime le besoin, jamais la solution.

SPEC

Spécification atomique (1 unité de livrable). Lie un Intent au code via `@spec SPEC-NNN-x` dans le code et `@verified-by` dans les tests.

SQS (Spec Quality Score)

Score 0..5 attribué par la Gate. ≥ 4/5 → la SPEC est prête à être exécutée par l'agent.

Execution Gate

Validation humaine de la SPEC avant exécution. Vérifie SQS + EARS lint + Test de l'Étranger.

Glossaire utile

Agent de gouvernance Tier 1

AI-ACT, RGPD, RGAA, RGESN, CRA. Ont un droit de veto sur le code/SPEC.

SBOM — Software Bill of Materials

Liste de tous les composants tiers (format CycloneDX). Exigé par le Cyber Resilience Act EU.

Score Souveraineté

Score composite 0..100 (Bronze→Platinum) qui mesure 5 dimensions : juridictions, agents Tier 1, langue, autorités, hébergement.

Tier 1

Niveau de gouvernance avec droit de veto. AIAD-AI-ACT, AIAD-RGPD, AIAD-RGAA, AIAD-RGESN, AIAD-CRA.

Glossaire utile

SPEC

Spécification atomique (1 unité de livrable). Lie un Intent au code via `@spec SPEC-NNN-x` dans le code et `@verified-by` dans les tests.

@verified-by

Annotation dans un fichier de test qui pointe vers la SPEC qu'il vérifie. Alimente la matrice backward.

EARS — Easy Approach to Requirements Syntax

Format de SPEC strict (WHEN/WHILE/IF/WHERE + sujet + SHALL). Activé via `format: EARS` en frontmatter.

SQS (Spec Quality Score)

Score 0..5 attribué par la Gate. ≥ 4/5 → la SPEC est prête à être exécutée par l'agent.
