INTENT-102-onboarding-mobile — Onboarding mobile en moins de 30 secondes

POURQUOI MAINTENANT

Hypothèse PM : les nouveaux comptes mobile (43 % des inscriptions du Q1) ne sont pas suffisamment guidés. Le drop-off à l'onboarding est de 38 %.

POUR QUI

Nouveaux utilisateurs mobiles iOS/Android, premier lancement de l'app.

OBJECTIF

Réduire le drop-off onboarding à < 20 % en 6 semaines.

CONTRAINTES

  • WCAG 2.1 AA (accessibilité)
  • Pas de SMS (RGPD + coût)

CRITÈRE DE DRIFT

Si drop-off > 30 % à la fin du sprint 5 → drift, revoir l'hypothèse PM plutôt que pousser une nouvelle itération.

SPECs liées

Aucune SPEC liée. Lance /sdd spec dans Claude Code pour décomposer cet Intent.

Risques

Niveau global : high

Réglementaire (mot-clé "rgpd") Accessibilité (mot-clé "wcag")

Hypothèse

untested

Si on simplifie l'onboarding mobile à 3 écrans (vs 5), le drop-off baisse de 38% à <20%

Dépendances

Bloqué par : aucune
