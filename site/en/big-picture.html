The cycle in text

Full alternative to the diagram above:

  1. Intent Statement (/sdd intent) — an identifiable human captures the WHY in five fields. Without a human author the session is invalidated (Human Authorship).
  2. Research Phase (/sdd research) — mandatory code-grounded discovery, graduated verdict GO / CONDITIONAL GO / DEFER / NO-GO decided by the human.
  3. SPEC (/sdd spec, /sdd split) — the living specification: scope, interface, behaviour, edge cases, tests, drift criterion. Prose or EARS format.
  4. Execution Gate (/sdd gate) — the SQS scores 5 criteria plus the Stranger Test. Below 4/5, the gate closes and the SPEC goes back to revision.
  5. Agent Execution (/sdd exec, /sdd resume) — optimised context within the Context Engineering Budget, implementation plan required before code.
  6. Validation (/sdd validate) — three axes: technical, functional, governance. The 4 Tier 1 agents hold a fail-closed veto.
  7. Drift Lock (/sdd drift-check, /sdd trace) — code and SPEC synchronised in the same PR, verified by the traceability matrix.
  8. Deployment — staging then production, monitoring, rollback if needed (Integrate loop).
  9. Consolidation (/aiad retro, /aiad intention, /aiad status) — Lessons Learned, Human Learnings, metrics. Then a new iteration from step 1 (rituals).

In permanent support: the artifacts (PRD, ARCHITECTURE, AGENT-GUIDE, Intents, SPECs), the five responsibilities, the JNSP uncertainty policy and the SQS, DORA and Flow metrics.
