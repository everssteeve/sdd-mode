About — the AIAD framework and its author

AIAD (Artificial Intelligence Agent Development) is an open-source meta-framework created by Steeve Evers and published at aiad.ovh. SDD Mode, which this site documents, is its Spec Driven Development operating mode.

What is AIAD?

AIAD is the organisational answer to an operational fact: AI agents have made the old way of building software obsolete. The question is no longer "how to code faster" but "how to think and orchestrate better". AIAD judges neither effort, nor velocity, nor process compliance — it judges the value realised for stakeholders.

The framework rests on seven immutable values — from the Primacy of Human Intent to Human Authorship — four pillars, and an ecosystem of five responsibilities rather than rigid roles.

SDD Mode

SDD Mode is the operational guide for AIAD Product Engineers: a strict cycle Intent → Research → SPEC → Gate → Execution → Validation → Drift Lock, tooled by the aiad-sdd npm package (33 slash commands, 4 governance agents, hooks, CI). See the Big Picture and the getting-started guide.

Positioning in the ecosystem

AIAD integrates the best SDD practices and adds the organisational and human dimension. Tools handle "how to work with agents"; AIAD handles "how to work as a human-AI team":

AIAD's positioning against SDD tools
ToolWhat it doesRelation to AIAD
Amazon KiroAgentic autonomy (autonomous mode, steering files)A possible execution layer for a validated AIAD SPEC — AIAD starts from intent, Kiro from execution.
GitHub Spec-KitSpecification productionA possible implementation of the meta-framework's spec layer.
SPDD (REASONS Canvas)Spec production methodThe REASONS Canvas is an option of /sdd spec; AIAD brings governance of the full cycle.
OpenAI SymphonyIssue-tracker-first intent artifactsConvergent on persistent intent; AIAD adds the SQS, the Drift Lock and regulatory governance.
Routines (Claude Code)Automated execution triggersCompatible: a Routine executes a SPEC already validated by the Gate, it does not replace it.

AIAD is model-agnostic by design: Intent Statements, SPECs and the Execution Gate are independent of the LLM and survive provider changes. Governance lives in the artifacts, not in the vendor. The 2026 industry validations (Context Engineering formalised by Anthropic, R2Code's −41.7% tokens, Thoughtworks Radar, AWS Strands) are independent confirmations of a prior approach.

Open source and contributing

AIAD applies its value 4, Radical Openness: methodological knowledge is shared, never locked away. The aiad-sdd package is published under the MIT licence (© 2026 Steeve Evers — AIAD Framework). Practices, tooling and field feedback are published at aiad.ovh; the package installs with npx aiad-sdd init (see the CLI reference).

To adopt the framework on an existing project: /aiad init bootstraps the structure from your real code, and /aiad onboard generates the briefing for newcomers.

This site, a demonstrator of the framework

This site applies the rules it documents: static HTML/CSS/JS with no framework, webfont or bitmap image (RGESN sobriety), keyboard navigation, AA contrast and text alternatives (RGAA), local search with no external service (GDPR). Its content is written from the repository's real artifacts — .aiad/, governance, commands, CHANGELOG — never invented.
