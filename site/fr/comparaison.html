Comparatif honnête — AIAD vs frameworks SDD

AIAD applique sa valeur Honnêteté sur les Contradictions : ce comparatif présente forces et faiblesses, sans masquer ce qui ne va pas. Les données mesurées par AIAD sont clairement séparées des données déclaratives issues de la documentation publique de chaque outil.

Légende
  • 🔬Mesuré Mesuré — benchmark exécuté sur machine de release (reproductible)
  • 📄Documenté Documenté — caractéristique déclarée dans la documentation officielle de l'outil
  • ❌Absent Absent — fonctionnalité non supportée (documenté)
  • N/D (YYYY-MM-DD) — donnée non disponible à cette date ; la fonctionnalité peut exister ou ne pas exister
  • N/A — non applicable à la nature de cet outil

Pourquoi AIAD se distingue

Le moat d'AIAD repose sur deux piliers que ses concurrents n'ont pas combinés :

Gouvernance réglementaire enforced

Quatre agents Tier 1 (AI Act, RGPD, RGAA, RGESN) ont un droit de veto sur chaque SPEC avant exécution. Ce n'est pas une checklist manuelle : la gate est automatisée et bloque le pipeline si un agent vote VETO. Aucun outil du marché à notre connaissance ne propose ce mécanisme de gouvernance réglementaire enforced à ce jour (2026-06-29).

La valeur concrète : un PE qui livre une feature touchant des données personnelles sans avoir déclenché la gate RGPD ne peut pas merger — le hook pre-commit bloque. La conformité n'est pas optionnelle.

Traçabilité machine-vérifiable

La matrice de traçabilité relie Intent Statement → SPEC → code → tests avec des annotations @intent, @spec, @verified-by vérifiées automatiquement. La GitHub Action sdd-trace.yml bloque toute PR créant un gap. Le Drift Lock garantit que code et SPEC restent synchrones dans la même PR.

Où AIAD est plus faible

Honnêteté sur les contradictions

AIAD applique sa valeur 2 à lui-même. Voici les faiblesses réelles au 2026-06-29 :

Couverture runtimes : 5 vs 30+ (Spec Kit)

AIAD émet des fichiers de règles pour 5 runtimes (Claude Code, Cursor, Codex, Copilot, Gemini). Spec Kit couvre 30+ runtimes. Si votre équipe utilise un agent IA non listé, AIAD ne génèrera pas de fichier de configuration natif — vous devrez contribuer le runtime ou maintenir le fichier manuellement.

L'extension des runtimes est inscrite à la roadmap (INTENT-023, SPEC-023-2 à venir). Les candidats prioritaires identifiés : Kiro, Amazon Q Developer.

Maturité des intégrations IDE

Kiro et Amazon Q sont intégrés directement dans l'IDE (VS Code, JetBrains) avec une UX native. AIAD est principalement CLI + hooks — l'intégration IDE passe par l'extension Claude Code ou Cursor, sans widget dédié ni panel AIAD. Pour les équipes qui travaillent exclusivement dans un IDE et préfèrent une UX graphique, Kiro offrira une expérience plus fluide.

Courbe d'apprentissage

Le cycle SDD (Intent → Research → SPEC → Gate → Exec → Validate → Drift Lock) impose une discipline plus exigeante que des outils qui démarrent à la SPEC ou directement à l'exécution. La valeur de cette rigueur apparaît sur des projets de 3 mois+ — sur un prototype en 2 jours, Kiro ou Cursor Memory Bank seront plus rapides à mettre en œuvre.
