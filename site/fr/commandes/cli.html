Ligne de commande

CLI npx aiad-sdd — installer et outiller

Les commandes slash vivent dans l'agent ; la CLI aiad-sdd vit dans le terminal et la CI. Elle installe le SDD Mode, le maintient à jour, et fournit les verdicts déterministes (traçabilité, parité multi-runtime, état du projet) qui transforment les rituels en mesures.

Installation en 1 minute

npx aiad-sdd init

Une seule commande installe dans votre projet : la structure .aiad/ (PRD, ARCHITECTURE, AGENT-GUIDE, intents, specs, métriques), les 4 agents de gouvernance Tier 1 (AI-ACT, RGPD, RGAA, RGESN), les commandes slash pour Claude Code et le fichier CLAUDE.md configuré. Puis, dans l'agent : /sdd init pour démarrer le cadrage.

Options principales de npx aiad-sdd init
OptionEffet
--sans-gouvernanceInitialise sans les agents de gouvernance Tier 1.
--minimalProfil AIAD-Lean : 4 commandes (intent / spec / gate / drift-check).
--upgrade <module>Ajoute un module au profil minimal : rituals, metrics, gouvernance ou all.
--runtime <list>Runtime IA ciblé : claude-code (défaut), cursor, codex, copilot, gemini, all.
--with-git-hooksInstalle le hook pre-commit (Drift Lock) en même temps.
--dry-run / --forceAperçu sans écrire / réinitialisation en écrasant l'existant.

npx aiad-sdd update — mettre à jour

Met à jour les commandes slash et les agents de gouvernance vers la version courante du package, en préservant vos fichiers : vos artefacts .aiad/ (PRD, Intents, SPECs, métriques) ne sont jamais touchés. Options : --sans-gouvernance pour ne pas toucher à la gouvernance, --dry-run pour l'aperçu, --check en CI (exit 1 si le projet diverge du package).

npx aiad-sdd status — état du projet

Affiche l'état SDD du projet directement dans le terminal : artefacts présents, cycle en cours, et score de maturité de 0 à 5 — de « non initialisé » à « complet ». Sorties --json (machine) et --markdown (collable dans une PR, Slack ou Notion).

npx aiad-sdd trace — traçabilité machine-vérifiable

Génère la matrice Intent ↔ SPEC ↔ Code ↔ Tests à partir des annotations @intent / @spec / @verified-by / @governance posées dans le code. C'est l'arbitre final du Drift Lock : la mesure remplace l'acte de foi.

npx aiad-sdd trace                 # matrice complète (md + json + html)
npx aiad-sdd trace --fail-on-gap   # CI : exit 1 si gap bloquant
npx aiad-sdd trace --json          # JSON brut sur stdout
Options de npx aiad-sdd trace
OptionEffet
--fail-on-gapExit 1 si un gap bloquant est détecté (SPEC orpheline, SPEC validée sans code) — usage CI.
--format <list>Formats produits : md, json, html (tous par défaut).
--out <dir>Dossier de sortie (défaut : .aiad/metrics/traceability/).
--watchRégénère la matrice à chaque changement (intent / spec / code).
--suggestCrée un squelette de SPEC pour chaque SPEC orpheline référencée par le code.

npx aiad-sdd emit-rules — multi-runtime

Régénère les fichiers de règles de tous les runtimes IA depuis la source amont unique (.aiad/AGENT-GUIDE.md + .aiad/gouvernance/) : AGENTS.md, le header de CLAUDE.md, .cursor/rules/*.mdc, .codex/AGENT.md et GEMINI.md. Une seule modification dans l'AGENT-GUIDE, et Claude Code, Cursor, Codex, Copilot et Gemini restent alignés.

npx aiad-sdd emit-rules                   # toutes les cibles
npx aiad-sdd emit-rules --runtime cursor  # un runtime précis
npx aiad-sdd emit-rules --check           # CI : exit 1 si divergence
Ne pas éditer les fichiers émis

Chaque fichier généré porte le header DO NOT EDIT et un source-hash SHA-256 : une modification manuelle sera écrasée à la prochaine génération, et détectée par le --check en CI. Toute règle se modifie dans l'AGENT-GUIDE ou la gouvernance, jamais dans une cible.

Exit codes : 0 / 1 / 2

La CLI suit une convention à trois états, alignée sur la politique d'incertitude JNSP du framework :

Convention d'exit codes de la CLI
CodeSémantique
0Succès / verdict PASS.
1Échec, erreur ou verdict FAIL.
2Verdict JNSP — décision humaine requise. Ce n'est pas une erreur : la commande a détecté une situation non décidable (intent flou, critère non testable, gouvernance ambiguë, fichier illisible) et refuse de trancher à la place de l'humain.

Ce troisième code permet à la CI de distinguer « bloqué en attente d'un humain » d'un vrai bug — il est notamment émis par les scorers déterministes du cycle (research, discovery-check, mini-gate).

Intégrations CI — GitHub Actions

Deux workflows fournis verrouillent les invariants du SDD Mode en pull request :

Workflows GitHub Actions fournis
WorkflowRôle
.github/workflows/sdd-trace.ymlInvoque aiad-sdd trace --fail-on-gap sur chaque PR — un gap bloquant de traçabilité fait échouer la pipeline (Drift Lock algorithmique).
.github/workflows/aiad-emit-rules-check.ymlInvoque aiad-sdd emit-rules --check — bloque toute PR où les fichiers de règles multi-runtime divergent de l'AGENT-GUIDE.

Hooks

La commande npx aiad-sdd hooks installe le hook Git pre-commit (et --uninstall le retire). Les hooks fournis dans .aiad/hooks/ ferment la boucle entre l'agent et le dépôt :

  • pre-commit.sh — bloque tout diff contenant un TODO-JNSP: non résolu et applique le Drift Lock avant commit. Le bypass git commit --no-verify existe mais est déconseillé.
  • session-start.js — exécuté à l'ouverture d'une session agent pour charger le contexte SDD initial.
  • discovery-gate.js — vérifie le prérequis Discovery (§3.5) : /sdd spec et /sdd exec ne démarrent pas sans une Research liée en verdict GO / CONDITIONAL GO.
Et bien plus

La CLI expose d'autres outils de support — doctor (diagnostic + score de santé), dashboard (le dashboard HTML décrit dans /aiad dashboard-html), gouvernance (mise à jour des agents Tier 1), uninstall… La référence exhaustive est disponible via npx aiad-sdd help.
