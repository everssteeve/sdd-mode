À propos — le framework AIAD et son auteur

AIAD (Artificial Intelligence Agent Development) est un méta-framework open source créé par Steeve Evers et publié sur aiad.ovh. Le SDD Mode, que ce site documente, en est le mode opératoire Spec Driven Development.

Qu'est-ce qu'AIAD ?

AIAD est la réponse organisationnelle à un constat opérationnel : les agents IA ont rendu obsolète la façon dont nous développions des logiciels. La question n'est plus « comment coder plus vite » mais « comment penser et orchestrer mieux ». AIAD ne juge ni sur l'effort, ni sur la vélocité, ni sur la conformité aux processus — il juge sur la valeur réalisée pour les stakeholders.

Le framework repose sur sept valeurs immuables — de la Primauté de l'Intention Humaine au Human Authorship — quatre piliers, et un écosystème de cinq responsabilités plutôt que de rôles rigides.

Le SDD Mode

Le SDD Mode est le guide opérationnel des Product Engineers AIAD : un cycle strict Intent → Research → SPEC → Gate → Exécution → Validation → Drift Lock, outillé par le paquet npm aiad-sdd (33 commandes slash, 4 agents de gouvernance, hooks, CI). Voir le Big Picture et le guide de démarrage.

Positionnement dans l'écosystème

AIAD intègre les meilleures pratiques SDD en leur ajoutant la dimension organisationnelle et humaine. Les outils gèrent le « comment faire avec les agents » ; AIAD gère le « comment travailler en équipe humain-IA » :

Positionnement d'AIAD face aux outils SDD
OutilCe qu'il faitRelation avec AIAD
Amazon KiroAutonomie agentique (mode autonomous, steering files)Couche exécution possible d'une SPEC AIAD validée — AIAD part de l'intention, Kiro de l'exécution.
GitHub Spec-KitProduction de spécificationsImplémentation possible de la couche spec du méta-framework.
SPDD (REASONS Canvas)Méthode de production de specLe REASONS Canvas est une option de /sdd spec ; AIAD apporte la gouvernance du cycle complet.
OpenAI SymphonyArtefacts d'intention issue-tracker-firstConvergence sur l'intention persistante ; AIAD ajoute SQS, Drift Lock et gouvernance réglementaire.
Routines (Claude Code)Déclencheurs d'exécution automatisésCompatibles : une Routine exécute une SPEC déjà validée par la Gate, elle n'en tient pas lieu.

AIAD est model-agnostic par conception : Intent Statements, SPECs et Execution Gate sont indépendants du modèle LLM et survivent aux changements de provider. La gouvernance est dans les artefacts, pas dans le vendor. Les validations industrielles 2026 (Context Engineering formalisé par Anthropic, R2Code −41,7 % de tokens, Thoughtworks Radar, AWS Strands) sont des confirmations indépendantes d'une approche antérieure.

Pour une comparaison factuelle forces et faiblesses assumées avec Spec Kit, Kiro, OpenSpec et BMAD, voir le comparatif honnête.

Open source et contribution

AIAD applique sa valeur 4, l'Ouverture Radicale : le savoir méthodologique se partage, il ne se verrouille pas. Le paquet aiad-sdd est publié sous licence MIT (© 2026 Steeve Evers — AIAD Framework). Pratiques, outils et retours d'expérience sont publiés sur aiad.ovh ; le paquet s'installe via npx aiad-sdd init (voir la référence CLI).

Pour adopter le framework sur un projet existant : /aiad init bootstrape la structure depuis votre code réel, et /aiad onboard génère le briefing des nouveaux membres.

Ce site, démonstrateur du framework

Ce site applique les règles qu'il documente : HTML/CSS/JS statique sans framework ni webfont ni image bitmap (sobriété RGESN), navigation clavier, contrastes AA et alternatives texte (RGAA), recherche locale sans service externe (RGPD). Son contenu est rédigé à partir des artefacts réels du dépôt — .aiad/, gouvernance, commandes, CHANGELOG — jamais inventé.
