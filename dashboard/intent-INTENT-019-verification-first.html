INTENT-019-verification-first — Verification-first — dériver des tests des critères EARS

POURQUOI MAINTENANT

Le marché SDD est passé de « comment écrire la spec » à « comment prouver que le code la respecte » : Kiro dérive des property-based tests directement des exigences EARS. AIAD valide la syntaxe EARS (ears-validator) mais n'en dérive rien d'exécutable. C'est le gap technique n°1 face à la concurrence.

POUR QUI

Les PE qui veulent une preuve de conformité, pas une promesse.

OBJECTIF

  • Générateur de squelettes de tests à partir d'une SPEC Format : EARS

validée (R1-R7), reliés par @verified-by.

  • trace --fail-on-gap couvre la chaîne EARS → test → code.

CONTRAINTES

  • EARS reste optionnel : ne rien imposer aux SPECs en prose.
  • Squelettes de test = point de départ, jamais une preuve auto-validée.

CRITÈRE DE DRIFT

Une SPEC EARS validée sans aucun test dérivé ni lié → drift verification-first.
