Search — explore the whole documentation

Instant search over the 28 pages of the SDD Mode documentation: type a term — SQS, drift, governance, EARS, JNSP — and jump straight to the right concept, command or glossary entry.

The search runs entirely in your browser against a static local index shipped with the site. No request leaves your machine, no query is logged, no third-party service is involved — the same privacy-by-design and digital-sobriety principles the framework enforces through its AIAD-RGPD and AIAD-RGESN governance agents. GDPR & RGESN friendly by construction.

    Suggested reading paths

    Not sure where to start? Three curated paths cover the most common entry points into the framework.

    Beginner — understand the cycle

    • Get Started — install SDD Mode and run your first cycle
    • Intent Statement — capture the human WHY before anything else
    • SPEC — turn the intent into an executable specification
    • Execution Gate & SQS — no code before the Gate opens (SQS ≥ 4/5)

    Practitioner — master the toolchain

    • /sdd commands — the 17 sub-commands of the SDD cycle router
    • Drift Lock — keep code and SPEC synchronised in every PR
    • Traceability — the machine-verifiable Intent ↔ SPEC ↔ Code ↔ Tests matrix

    Compliance — govern with confidence

    • Tier 1 governance — AI-ACT, RGPD, RGAA and RGESN agents with veto power
    • JNSP — uncertainty policy — "I don't know" as a valid, fail-closed signal
    • Validation — verify the delivered code against SPEC and governance
    Prefer asking your agent?

    /aiad-help <keyword> performs the same lookup directly inside Claude Code and routes you to the matching command or concept.
