The heart of the framework

One cycle, seven steps, zero code without a validated spec

Development follows a strict cycle — no step is ever skipped. Each step produces a traceable artifact, from the human WHY down to the anti-drift lock on the PR.

  1. 1. Intent Statement

    The human intent in 5 fields: why now, for whom, objective, constraints, drift criterion. A human author is mandatory.

    Read the article →

  2. 2. Research

    Code-grounded discovery, graduated verdict GO / CONDITIONAL GO / DEFER / NO-GO — decided by the human.

    Read the article →

  3. 3. SPEC

    The living specification: scope, interface, behaviour, edge cases, tests. Prose format or strict EARS.

    Read the article →

  4. 4. Execution Gate

    The SQS scores the SPEC on 5 criteria plus the Stranger Test. Gate closed below 4/5 — no agent starts.

    Read the article →

  5. 5. Agent Execution

    Optimised context (AGENT-GUIDE + condensed ARCHITECTURE + a single SPEC), implementation plan before code.

    Read the article →

  6. 6. Validation

    Three axes: technical, functional, governance. The 4 Tier 1 agents hold a fail-closed veto.

    Read the article →

  7. 7. Drift Lock

    Code and SPEC synchronised in the same PR. Drift is a process failure, not an agent error.

    Read the article →

Foundations

Three founding principles

  • Spec as Living Invariant

    The SPEC is not a transit document: it remains the source of truth between human intent and agent code — before, during and after implementation. AIAD is spec-anchored: neither throwaway spec-first nor spec-as-source.

    The living SPEC →

  • Drift = process failure

    Code that evolves without its artifacts is not done. Updating the SPEC is part of the Definition of Done — verified algorithmically by the traceability matrix.

    The Drift Lock →

  • Context Engineering Budget

    The Product Engineer owns the context budget of every session: at most 60-70% of the window, a single SPEC at a time, the Intent at the top of the context.

    The Context Budget →

Tier 1 — veto power

Regulatory governance built into the cycle

Four governance agents are injected into every session that touches their scope. Verdict UNKNOWN = VETO: the policy is fail-closed — never silent self-clearance.

  • AIAD-AI-ACT

    Regulation (EU) 2024/1689. Risk classification and obligations whenever an AI component is involved — ML, LLM, scoring, recommendation.

    Learn more →

  • AIAD-RGPD

    Privacy by Design: minimisation, legal bases, data-subject rights — whenever the code processes personal data.

    Learn more →

  • AIAD-RGAA

    Digital accessibility RGAA 4.1 / WCAG 2.1 — whenever the code produces a user interface.

    Learn more →

  • AIAD-RGESN

    RGESN v2 eco-design: computational sobriety on every technical decision — performance, resources, dependencies.

    Learn more →

The human ecosystem

AIAD Constitution

Seven immutable values

They are not aspirations — they are constitutive of the framework. Remove one and the whole collapses.

  1. 1. Primacy of Human Intent

    The agent executes. It does not decide what deserves to be built.

  2. 2. Honesty about Contradictions

    Tell the truth about trade-offs rather than promise the impossible.

  3. 3. Intentional Sobriety

    Build what is necessary, not what is impressive.

  4. 4. Radical Openness

    Open source: methodological knowledge is shared, never locked away.

  5. 5. Uncompromising Empiricism

    Validate through observation, not consensus.

  6. 6. Shared Responsibility

    Success and failure belong to the team — never to blame.

  7. 7. Human Authorship

    Authorship of intent cannot be delegated. If the agent wrote the Intent, the session is invalidated.
