What's New — the evolution of SDD Mode

Version by version, how the framework moved from a documented method to an enforced toolchain — newest first, summarised from the official aiad-sdd changelog. Releases follow Keep a Changelog and SemVer.

v1.18 — advisory becomes enforced (June 2026)

The major shift: every critical SDD rule becomes a primitive verified by the harness, and every verdict is computed by a deterministic script (exit codes 0/1/2 plus validated JSON) — never by the model's free judgement. Entirely additive and opt-in. Highlights:

  • Enforced governance — PreToolUse hooks for JNSP and the fail-closed Tier 1 veto, a Stop hook for the Drift Lock, read-only governance subagents. The Tier 1 veto cannot be disabled by an environment toggle.
  • Research Phase — /sdd research with the graduated GO | CONDITIONAL GO | DEFER | NO-GO verdict, anchored in mandatory Discovery. See Research Phase.
  • Phased execution — mini-gates per slice and a third CONDITIONAL verdict; crash-recoverable cycle as a task graph; memory "from logs" with human-authored curation.
  • Design guardrails — proportionality, the interactive grill-me gate, sunset metadata, and a canary suite separating real regressions from model-serving noise.

v1.16 / v1.15 — the PM cockpit (June 2026)

Two consecutive releases (v1.17 shipped no user-visible changes) massively expanded the HTML dashboard into a Product Manager cockpit: Kanban with role-aware lenses, OKR alignment, DORA/flow views, risk burndown, stakeholder maps, sprint planning, AI-Act compliance views and dozens of focused widgets. Also added: a practical SDD-Mode guide, a qualitative feedback command, and 43 new OpenAPI routes.

v1.14 — the European compliance toolchain (May 2026)

46 items across two phases, 323 automated tests, zero runtime dependencies preserved. Key additions:

  • aiad-sdd ai-act audit — pre-fills the Annex IV documentation of EU Regulation 2024/1689 across its 8 required sections.
  • Jurisdiction governance packs (eu-baseline, us-baseline, uk-baseline) and a community pack marketplace with SHA-256 validation.
  • doctor, uninstall, migrate, workspace, repl, opt-in GDPR-compliant telemetry, and a SARIF output for trace consumable by GitHub Code Scanning.
  • Multi-language annotations (Rust, Go, Java, Kotlin, C#, Ruby, PHP, Swift, Scala, Elixir), --watch and --suggest modes, and Git-aware scanning (~1.8 s on 100k files).

v1.13 — HTML dashboard (May 2026)

First HTML dashboard, the --serve mode and a GitHub Pages workflow.

v1.12 — multi-runtime (May 2026)

aiad-sdd emit-rules generates AGENTS.md, .cursor/rules/, .codex/AGENT.md and GEMINI.md from a single source — the AGENT-GUIDE — so Claude Code, Cursor, Copilot, Codex and Gemini all follow the same rules. A CI check blocks any divergence in PRs.

v1.11 — optional EARS variant (May 2026)

SPECs can be written in EARS (Easy Approach to Requirements Syntax) with a strict R1–R7 linter at the Gate: zero violations earns a guaranteed +1 on the SQS Testability criterion. Reserved for SPECs where ambiguity is expensive. See SPEC — EARS variant.

v1.10 — machine-verifiable traceability (April 2026)

The Intent ↔ SPEC ↔ Code ↔ Tests matrix built from @intent / @spec / @verified-by / @governance annotations, generated by aiad-sdd trace and enforced in CI with --fail-on-gap. See Traceability.
