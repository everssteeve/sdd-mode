Command line

npx aiad-sdd CLI — install and tool up

The slash commands live in the agent; the aiad-sdd CLI lives in the terminal and the CI. It installs SDD Mode, keeps it up to date, and provides the deterministic verdicts (traceability, multi-runtime parity, project state) that turn rituals into measurements.

1-minute install

npx aiad-sdd init

A single command installs in your project: the .aiad/ structure (PRD, ARCHITECTURE, AGENT-GUIDE, intents, specs, metrics), the 4 Tier 1 governance agents (AI-ACT, RGPD, RGAA, RGESN), the slash commands for Claude Code and the configured CLAUDE.md file. Then, in the agent: /sdd init to start the framing.

Main options of npx aiad-sdd init
OptionEffect
--sans-gouvernanceInitialise without the Tier 1 governance agents.
--minimalAIAD-Lean profile: 4 commands (intent / spec / gate / drift-check).
--upgrade <module>Adds a module to the minimal profile: rituals, metrics, gouvernance or all.
--runtime <list>Target AI runtime: claude-code (default), cursor, codex, copilot, gemini, all.
--with-git-hooksInstalls the pre-commit hook (Drift Lock) at the same time.
--dry-run / --forcePreview without writing / reinitialise by overwriting the existing setup.

npx aiad-sdd update — update

Updates the slash commands and the governance agents to the current version of the package, while preserving your files: your .aiad/ artifacts (PRD, Intents, SPECs, metrics) are never touched. Options: --sans-gouvernance to leave governance untouched, --dry-run for a preview, --check in CI (exit 1 if the project diverges from the package).

npx aiad-sdd status — project state

Displays the project's SDD state directly in the terminal: artifacts present, current cycle, and a maturity score from 0 to 5 — from "not initialised" to "complete". Outputs --json (machine) and --markdown (pasteable into a PR, Slack or Notion).

npx aiad-sdd trace — machine-verifiable traceability

Generates the Intent ↔ SPEC ↔ Code ↔ Tests matrix from the @intent / @spec / @verified-by / @governance annotations placed in the code. It is the final arbiter of the Drift Lock: measurement replaces an act of faith.

npx aiad-sdd trace                 # full matrix (md + json + html)
npx aiad-sdd trace --fail-on-gap   # CI: exit 1 on a blocking gap
npx aiad-sdd trace --json          # raw JSON on stdout
Options of npx aiad-sdd trace
OptionEffect
--fail-on-gapExit 1 if a blocking gap is detected (orphan SPEC, validated SPEC without code) — CI usage.
--format <list>Produced formats: md, json, html (all by default).
--out <dir>Output folder (default: .aiad/metrics/traceability/).
--watchRegenerates the matrix on every change (intent / spec / code).
--suggestCreates a SPEC skeleton for every orphan SPEC referenced by the code.

npx aiad-sdd emit-rules — multi-runtime

Regenerates the rule files of all the AI runtimes from the single upstream source (.aiad/AGENT-GUIDE.md + .aiad/gouvernance/): AGENTS.md, the CLAUDE.md header, .cursor/rules/*.mdc, .codex/AGENT.md and GEMINI.md. One single change in the AGENT-GUIDE, and Claude Code, Cursor, Codex, Copilot and Gemini stay aligned.

npx aiad-sdd emit-rules                   # all targets
npx aiad-sdd emit-rules --runtime cursor  # one specific runtime
npx aiad-sdd emit-rules --check           # CI: exit 1 on divergence
Do not edit the emitted files

Each generated file carries the DO NOT EDIT header and a SHA-256 source-hash: a manual modification will be overwritten at the next generation, and detected by --check in CI. Every rule is modified in the AGENT-GUIDE or the governance, never in a target.

Exit codes: 0 / 1 / 2

The CLI follows a three-state convention, aligned with the framework's JNSP uncertainty policy ("Je Ne Sais Pas" — I don't know):

CLI exit code convention
CodeSemantics
0Success / PASS verdict.
1Failure, error or FAIL verdict.
2JNSP verdict — human decision required. It is not an error: the command detected an undecidable situation (vague intent, untestable criterion, ambiguous governance, unreadable file) and refuses to decide in place of the human.

This third code lets the CI distinguish "blocked waiting for a human" from a real bug — it is notably emitted by the cycle's deterministic scorers (research, discovery-check, mini-gate).

CI integrations — GitHub Actions

Two provided workflows lock down the SDD Mode invariants on pull requests:

Provided GitHub Actions workflows
WorkflowRole
.github/workflows/sdd-trace.ymlInvokes aiad-sdd trace --fail-on-gap on every PR — a blocking traceability gap fails the pipeline (algorithmic Drift Lock).
.github/workflows/aiad-emit-rules-check.ymlInvokes aiad-sdd emit-rules --check — blocks any PR where the multi-runtime rule files diverge from the AGENT-GUIDE.

Hooks

The npx aiad-sdd hooks command installs the Git pre-commit hook (and --uninstall removes it). The hooks provided in .aiad/hooks/ close the loop between the agent and the repository:

  • pre-commit.sh — blocks any diff containing an unresolved TODO-JNSP: and applies the Drift Lock before commit. The git commit --no-verify bypass exists but is discouraged.
  • session-start.js — executed when an agent session opens, to load the initial SDD context.
  • discovery-gate.js — verifies the Discovery prerequisite (§3.5): /sdd spec and /sdd exec do not start without a linked Research at a GO / CONDITIONAL GO verdict.
And much more

The CLI exposes other supporting tools — doctor (diagnosis + health score), dashboard (the HTML dashboard described in /aiad dashboard-html), gouvernance (Tier 1 agent updates), uninstall… The exhaustive reference is available via npx aiad-sdd help.
