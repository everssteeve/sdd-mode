Nouveautés — l'évolution du SDD Mode

Le résumé version par version du CHANGELOG officiel du paquet aiad-sdd, de la plus récente à la plus ancienne. Chaque version est reliée aux articles du site qui documentent les fonctionnalités concernées.

v1.18 — « advisory → enforced » (juin 2026)

Release majeure : chaque règle critique du SDD devient une primitive vérifiée par le harness, et chaque verdict est calculé par un script déterministe (exit 0/1/2 + JSON validé), jamais par le jugement libre du modèle. Tout est additif et opt-in — aucune rupture.

  • Gouvernance enforced : hooks PreToolUse (JNSP, veto Tier 1 fail-closed) et Stop (Drift Lock), subagents de gouvernance read-only, veto non désactivable par un simple toggle.
  • Phase Research : verdict gradué GO | CONDITIONAL GO | DEFER | NO-GO ancré dans la Discovery, intercalé entre l'Intent et la SPEC.
  • Exécution phasée : mini-gates par tranche et troisième verdict CONDITIONAL — voir Exécution agent.
  • Mémoire, cycle, canary, observabilité : mémoire « from logs » auto-curée, cycle SDD matérialisé en graphe de tâches, suite canary anti-bruit de serving, statusLine live.
  • Garde-fous : proportionnalité, gate interactif grill-me, métadonnée d'obsolescence sunset, review cross-model additive-only.

v1.15 → v1.17 (juin 2026)

Trois versions de consolidation produit (la v1.17 ne contient pas de changement utilisateur). L'essentiel :

  • Dashboard PM cockpit : des dizaines de modules de pilotage produit (couverture PRD, vélocité, OKR, Kanban role-aware, scorecards, forecast) — voir Métriques — dashboards.
  • Guide pratique SDD Mode + score de santé du backlog, commande de feedback qualitatif opt-in.
  • Trace : parsing frontmatter, cache, worker threads, mode watch, multi-langage — voir Traçabilité.
  • Assistants /sdd prd et /sdd arch + plan de parallélisme des SPECs — voir Commandes /sdd.

v1.14 — fondations EU et écosystème (mai 2026)

46 items livrés pour positionner aiad-sdd comme framework européen de développement basé sur l'intention — 323 tests automatisés, zéro dépendance runtime.

  • ai-act audit : pré-remplit la documentation Annexe IV du Règlement (UE) 2024/1689 — voir AIAD-AI-ACT.
  • Packs de gouvernance par juridiction (eu-baseline, us-baseline, uk-baseline) et marketplace de packs signés SHA-256 — voir Gouvernance — packs.
  • Trace ++ : --watch, --suggest, annotations dans 13 langages, export SARIF v2.1.0, scan via git ls-files (100 000 fichiers en ~1,8 s).
  • Outillage : doctor, workspace multi-projet, REPL, migrate, télémétrie opt-in RGPD-compliant, i18n FR/EN — voir CLI npx aiad-sdd.
  • Écosystème : GitHub Action officielle, templates GitLab CI / Bitbucket / Drone, extension VS Code, site de documentation.

v1.13 — dashboard HTML (mai 2026)

  • Dashboard HTML multi-pages, mode --serve et workflow GitHub Pages — voir Métriques et /aiad dashboard-html.

v1.12 — multi-runtime (mai 2026)

  • aiad-sdd emit-rules fait d'AIAD la source amont des règles agent : .aiad/AGENT-GUIDE.md + gouvernance génèrent AGENTS.md, .cursor/rules/, .codex/AGENT.md et GEMINI.md — voir CLI — emit-rules. Le workflow CI aiad-emit-rules-check.yml bloque toute divergence.

v1.11 — variante EARS (mai 2026)

  • Rédaction optionnelle des SPECs au format EARS (/sdd spec --ears) avec linter strict R1–R7 : 0 violation garantit +1 sur le critère SQS Testabilité — voir SPEC — variante EARS. À réserver aux SPECs où l'ambiguïté coûte cher (sécurité, paiement, conformité).

v1.10 — traçabilité machine-vérifiable (avril 2026)

  • Matrice Intent ↔ SPEC ↔ Code ↔ Tests construite depuis les annotations @intent, @spec, @verified-by, @governance, générée par /sdd trace ou npx aiad-sdd trace — voir Traçabilité. La GitHub Action sdd-trace.yml fait échouer la PR sur gap bloquant.
