Le Big Picture SDD Mode

Tout le mode opératoire sur un seul schéma : le cycle en sept étapes, la gouvernance Tier 1 qui le surplombe, les artefacts qui l'alimentent, les responsabilités qui le portent et les métriques qui le mesurent. Cliquez sur n'importe quel élément pour ouvrir l'article correspondant.

Gouvernance Tier 1 (veto) Étapes du cycle Gates & Human Authorship Artefacts & Drift Lock Responsabilités Rituels & métriques

Le cycle en texte

Alternative complète au schéma ci-dessus :

  1. Intent Statement (/sdd intent) — un humain identifiable capture le POURQUOI en cinq champs. Sans auteur humain, la session est invalidée (Human Authorship).
  2. Phase Research (/sdd research) — discovery obligatoire ancrée dans le code, verdict gradué GO / CONDITIONAL GO / DEFER / NO-GO tranché par l'humain.
  3. SPEC (/sdd spec, /sdd split) — spécification vivante : scope, interface, comportement, cas limites, tests, critère de drift. Format prose ou EARS.
  4. Execution Gate (/sdd gate) — le SQS score 5 critères + le Test de l'Étranger. Sous 4/5, la gate ferme et la SPEC retourne en révision.
  5. Exécution agent (/sdd exec, /sdd resume) — contexte optimisé dans le Context Engineering Budget, plan d'implémentation exigé avant le code.
  6. Validation (/sdd validate) — trois axes : technique, fonctionnel, gouvernance. Les 4 agents Tier 1 ont un droit de veto fail-closed.
  7. Drift Lock (/sdd drift-check, /sdd trace) — code et SPEC synchronisés dans la même PR, vérifié par la matrice de traçabilité.
  8. Déploiement — staging puis production, monitoring, rollback si besoin (boucle Intégrer).
  9. Consolidation (/aiad retro, /aiad intention, /aiad status) — Lessons Learned, Human Learnings, métriques. Puis nouvelle itération à partir de l'étape 1 (rituels).

En soutien permanent : les artefacts (PRD, ARCHITECTURE, AGENT-GUIDE, Intents, SPECs), les cinq responsabilités, la politique d'incertitude JNSP et les métriques SQS, DORA et Flow.
